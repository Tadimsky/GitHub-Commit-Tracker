5 
6 MEAN is a boilerplate that provides a nice starting point for [MongoDB](http://www.mongodb.org/), [Node.js](http://www.nodejs.org/), [Express](http://expressjs.com/), and [AngularJS](http://angularjs.org/) based applications. It is designed to give you a quick and organized way to start developing MEAN based web apps with useful modules like Mongoose and Passport pre-bundled and configured. We mainly try to take care of the connection points between existing popular frameworks and solve common integration problems.

34 ## Quick Install
35  The quickest way to get started with MEAN is to install the `mean-cli` package from NPM.
36 
37  Install MEAN CLI:
38 
39  $ [sudo] npm install -g mean-cli
40  $ mean init <myApp>
41  $ cd <myApp> && npm install
42 
43  We recommend using [Grunt](https://github.com/gruntjs/grunt-cli) to start the server:
44 
45  $ grunt
46 
47  If grunt aborts because of JSHINT errors, these can be overridden with the `force` flag:
48 
49  $ grunt -f
50 
51  Alternatively, when not using `grunt` you can run:
52 
53  $ node server
54 
55  Then, open a browser and go to:
56 
57  http://localhost:3000

107 ## Configuration
108 All configuration is specified in the [config](/config/) folder, through the [env](config/env/) files, and is orchestrated through the [meanio](https://github.com/linnovate/mean-cli) NPM module. Here you will need to specify your application name, database name, and hook up any social app keys if you want integration with Twitter, Facebook, GitHub, or Google.
109 
110 ### Environmental Settings
111 
112 There is a shared environment config: __all__.
113 * __root__ - This the default root path for the application.
114 * __port__ - DEPRECATED to __http.port__ or __https.port__.
115 * __http.port__ - This sets the default application port.
116 * __https__ - These settings are for running HTTPS / SSL for a secure application.
117  * __port__ - This sets the default application port for HTTPS / SSL. If HTTPS is not used then is value is to be set to __false__ which is the default setting. If HTTPS is to be used the standard HTTPS port is __443__.
118  * __ssl.key__ - The path to public key.
119  * __ssl.cert__ - The path to certificate.
120 
121 There are three environments provided by default: __development__, __test__, and __production__.
122 
123 Each of these environments has the following configuration options:
124 
125 * __db__ - This is where you specify the MongoDB / Mongoose settings
126  * __url__ - This is the url/name of the MongoDB database to use, and is set by default to __mean-dev__ for the development environment.
127  * __debug__ - Setting this option to __true__ will log the output all Mongoose executed collection methods to your
128 console. The default is set to __true__ for the development environment.
129  * __options__ - These are the database options that will be passed directly to mongoose.connect in the __production__ environment: [server, replset, user, pass, auth, mongos] (http://mongoosejs.com/docs/connections.html#options) or read [this] (http://mongodb.github.io/node-mongodb-native/driver-articles/mongoclient.html#mongoclient-connect-options) for more information.
130 * __app.name__ - This is the name of your app or website, and can be different for each environment. You can tell which environment you are running by looking at the TITLE attribute that your app generates.
131 * __Social OAuth Keys__ - Facebook, GitHub, Google, Twitter. You can specify your own social application keys here for each platform:
132  * __clientID__
133  * __clientSecret__
134  * __callbackURL__
135 * __emailFrom__ - This is the from email address displayed when sending an email.
136 * __mailer__ - This is where you enter your email service provider, username and password.
137 
138 To run with a different environment, just specify NODE_ENV as you call grunt:
139 
140  $ NODE_ENV=test grunt
141 
142 If you are using node instead of grunt, it is very similar:
143 
144  $ NODE_ENV=test node server
145 
146 To simply run tests
147 
148  $ npm test
149 
150 > NOTE: Running Node.js applications in the __production__ environment enables caching, which is disabled by default in all other environments.

152 ## Maintaining your own repository
153 After initializing a project, you'll see that the root directory of your project is already a git repository. MEAN uses git to download and update its own code. To handle its own operations, MEAN creates a remote called `upstream`. This way you can use git as you would in any other project.
154 
155 To maintain your own public or private repository, add your repository as remote. See here for information on [adding an existing project to GitHub](https://help.github.com/articles/adding-an-existing-project-to-github-using-the-command-line).
156 
157 ```
158 git remote add origin <remote repository URL>
159 git push -u origin master
160 ```

174 ## Heroku Quick Deployment
175 Before you start make sure you have the [Heroku toolbelt](https://toolbelt.heroku.com/)
176 installed and an accessible MongoDB instance - you can try [MongoHQ](http://www.mongohq.com/)
177 which has an easy setup).
178 
179 Add the db string to the production env in server/config/env/production.js.
180 
181 ```
182 git init
183 git add .
184 git commit -m "initial version"
185 heroku apps:create
186 heroku config:add NODE_ENV=production
187 heroku config:add BUILDPACK_URL=https://github.com/mbuchetics/heroku-buildpack-nodejs-grunt.git
188 git push heroku master
189 heroku config:set NODE_ENV=production
190 ```
